# 页面职责与导航规范

> 当前主文档已迁移到 [docs/family-ai-os/03-page-navigation.md](family-ai-os/03-page-navigation.md)。
>
> 本文件作为历史兼容入口保留，后续页面职责调整优先修改 `docs/family-ai-os/03-page-navigation.md`。

本文是 Family AI OS 圆屏端的页面落位基准。目标不是增加页面数量，而是让每个入口有稳定含义：用户知道去哪，开发时也知道功能应该放在哪一层。

## 产品定位

设备定位为全家可用的 AI 圆屏中控终端：

- 平时是时间、天气、日程、音乐、相册和轻状态屏。
- 需要语音时是小智官方高速 AI 入口。
- 孩子使用时是英语练习、故事、小游戏和学习记录入口。
- 家庭使用时是 OpenClaw、Home Assistant、服务器音乐、家庭相册和家长后台的前端触点。

ESP32 负责稳定渲染、触摸、音频、SD 资源和离线兜底；服务端负责业务数据、内容资源、记忆、资源包、权限和扩展程序调用。注意：“内容系统”是后端资源层概念，不再作为圆屏端一级页面名称。

## 全局导航

首页保持当前实现，不在本轮重构。

顶部系统栏固定承担系统入口：

- 时间
- 首页
- 当前短标题
- 后端状态
- 设置

页面主体区只放当前页面的主要内容。

页面操作栏只放当前页面业务动作，例如播放、暂停、下一首、确认、返回、刷新、扫描。

底部轻状态栏只显示一行轻状态、toast、同步结果或后端动作反馈；不承载 AI 正文。

## 页面分层

### 系统级入口

系统级入口不属于应用页：

- 首页
- 设置
- 顶部后端状态
- 全局 AI 唤醒词

设置只从顶部系统栏进入。应用页、Remote UI 应用列表和家庭工具页都不应该放设置入口。

### 一级能力页

一级能力页是用户心智里的主要功能：

- AI
- 天气
- 日程
- 音乐
- 英语
- 相册
- 应用
- 设置

注意：一级能力不等于都必须在当前首页展示。首页保持现状；缺少首页入口的一级能力可以先通过页面轮转、过渡入口或 Remote UI 进入，后续产品评审后再决定是否调整首页。

### 扩展应用页

应用页只承载可扩展应用和家庭工具：

- 小游戏
- OpenClaw
- Home Assistant 场景
- 家庭工具
- 实验功能

应用页不放设置、天气、音乐、英语、相册这类已经有明确页面职责的系统或一级能力。

## 页面职责

| 页面 | 职责 | 不放什么 | 主要入口 | 离线兜底 |
| --- | --- | --- | --- | --- |
| 首页 | 稳定首屏、时间/天气摘要、下一日程、AI/音乐/英语/应用快捷入口 | 新复杂布局、深层设置、长文本 | 启动、顶部首页按钮 | 必须本地可用 |
| AI | 小智状态、唤醒/听说状态、AI 对话视觉反馈、工具调用提示 | 家庭内容列表、设置项、长篇历史记录 | 首页 AI、唤醒词、任意页自动浮层 | 官方小智仍可用 |
| 天气 | 当前天气、温度、空气、湿度、预报摘要、刷新 | 日程事项、复杂气象报表、设置入口 | 首页天气按钮 | 显示最近缓存/本地占位 |
| 日程 | 今日安排、下一项、完成、稍后提醒 | 相册、应用列表、系统诊断 | 首页下一项、通知页 | 显示缓存和离线待同步 |
| 音乐 | SD 本地音乐与服务器播客/故事音频双通道播放控制 | 英语练习流程、相册浏览、系统设置 | 首页音乐 | SD 音乐可本地播放 |
| 英语 | 口语跟读、AI 对话练习、单词/句子练习、学习记录 | 通用内容库管理、系统设置 | 首页英语 | 本地提示和待同步记录 |
| 相册 | 家庭照片浏览、幻灯片播放、屏保资源选择、SD/服务器相册缓存状态 | 故事、播客、英语课程、小游戏、设备设置 | 页面轮转/后续首页入口/Remote UI | SD 图片和屏保可本地显示 |
| 应用 | 小游戏、OpenClaw、Home Assistant、家庭工具、实验功能的扩展中心 | 设置、天气、音乐、英语、相册等一级入口重复项 | 首页应用 | 本地小游戏可运行 |
| 设置 | 网络、蓝牙、USB、蜂窝预留、电源、存储、系统、诊断 | 业务应用、小游戏、内容列表 | 顶部设置按钮 | 必须本地可用 |

## 页面操作栏规则

页面操作栏是页面自己的工具条，不是全局导航栏。

- 音乐页：播放、暂停、下一首/下一集、音量、扫描。
- AI 页：开始说话、停止/打断、返回。
- 日程页：完成、稍后、返回。
- 设置详情页：刷新、返回设置首页。
- 应用子页：返回应用、首页。
- 相册页：上一张、下一张、设为屏保、开始/停止幻灯片。
- 屏保页：退出。

当页面已经能通过顶部系统栏进入设置时，页面操作栏不再放“设置”。

## Remote UI 规则

Remote UI 可以下发应用页、英语页、相册页等配置，但必须遵守页面职责：

- 不执行脚本、HTML、CSS 或远程代码。
- 只渲染白名单组件。
- 只执行白名单动作。
- 不下发设置作为应用项。
- 不把相册、天气、音乐、英语等一级页面伪装成应用项。
- `app.open(settings)` 是保留路由，设备端应拒绝或提示使用顶部设置入口。
- 未知组件、未知 action、超长字段、资源缺失必须安全降级。

## 后端职责边界

后端可以改变：

- 应用列表中的扩展应用。
- 后端内容系统中的课程包、相册包、播客包、故事包和小游戏资源包。
- 相册页可见的相册、照片、封面、屏保资源和幻灯片配置。
- 英语练习主题、提示语、推荐内容。
- 天气、日程、音乐、通知数据。
- OpenClaw/Home Assistant 工具配置。

后端不应该改变：

- 设置入口归属。
- 顶部系统栏基本结构。
- 首页在当前阶段的稳定布局。
- 本地兜底页面可用性。

## 下一步落位顺序

1. 固化当前首页，不做视觉和入口大改。
2. 检查天气页，移除日程事项和日程按钮，只保留天气信息与刷新。
3. 将现有 Content 路由迁移为相册页，只承担家庭照片浏览和屏保资源。
4. 检查应用页，只保留扩展应用、小游戏和家庭工具。
5. 检查英语页，把练习流程和学习记录放进去，不做通用内容管理。
6. 检查音乐页，继续保持 SD 音乐和服务器播客/故事音频双通道。
7. 检查设置页，补齐网络、蓝牙、电源、存储、系统、诊断细节。
8. 检查 Remote UI，保证服务端下发页面不破坏上述边界。

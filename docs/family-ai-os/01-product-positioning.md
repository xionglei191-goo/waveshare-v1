# 产品定位

Family AI OS 不是一个“功能尽量塞进 ESP32 的设备项目”，而是一个全家共享的家庭 AI 交互生态。

核心形态：

```text
圆屏交互端 + AI-native 后端 + 手机 App 预留
```

一句话定位：

> 一个以圆屏为家庭现场入口、以后端为 AI 能力大脑、以手机为便携管理和连接桥的家庭 AI 交互系统。

## 核心价值

圆屏不和手机比功能多。它的价值是简单、方便、自然。

- 固定在家里，不需要找设备。
- 全家共享，不绑定某一个人的手机。
- 低打扰，不把用户带进手机通知和娱乐流。
- 适合短交互：看一眼、点一下、说一句、听反馈、确认/取消。
- 适合家庭现场：厨房、床头、客厅、书桌和儿童活动区。

判断功能是否适合圆屏的规则：

> 这个功能能否在家庭现场 3 秒内完成主要交互？

如果需要复杂输入、长列表、长文本、账号管理或内容维护，应放到后端管理台或手机 App。

## 三端职责

### 圆屏

圆屏是家庭现场交互端。

负责：

- 展示时间、天气、日程、播放状态、AI 状态和设备状态。
- 触摸、语音入口、简单按钮、确认/取消。
- 本地音频播放、服务器音频播放、提示音和简单动画。
- 接收后端设备命令并 ACK。
- 后端不可用时提供本地兜底页面和基础操作。

不负责：

- 搜索、推荐、权限判断、内容管理、工具编排。
- 长列表浏览、复杂表单、后台配置。
- 大模型推理、知识库检索、家庭记忆维护。
- 运行 HTML/CSS/JS 或任意服务端脚本。

### 后端

后端是 AI-native Family Capability Hub。

负责：

- 媒体库、播客、相册、课程、小游戏资源。
- 家庭成员、角色、权限、家庭模式和审计。
- 日程、通知、记忆、学习记录和知识库。
- OpenClaw、Home Assistant、NAS 等家庭工具。
- Remote UI 配置、设备命令队列、同步和诊断。
- 给小智官方、自建 AI、手机 App、后台和圆屏提供统一能力接口。

原则：

> 后端所有核心能力都要 AI-native，但不能把所有 API 直接暴露给 AI；AI 只能调用经过语义封装、权限校验和审计的安全工具。

### 手机 App

手机 App 是便携管理和连接桥，第一阶段预留，不抢圆屏现场交互角色。

负责：

- 配网、绑定、诊断和日志查看。
- 内容导入、资源管理、家庭成员和权限配置。
- 外出访问家庭状态。
- 无 Wi-Fi 场景下作为 BLE/手机网络代理。
- 后续承接复杂管理体验。

## 与小智的关系

小智官方继续保留为高速语音入口：

- 唤醒词。
- ASR/TTS 和低延迟对话。
- 官方 OTA/MQTT 和音频链路。
- 通过 MCP/工具调用 Family Backend。

小智不是长期家庭大脑。长期家庭智能由自建 AI 服务承担：

- 多用户角色。
- 多重记忆。
- 性格画像。
- 家庭知识库。
- 权限策略。
- Agent 编排。

## 产品原则

- 圆屏轻量，后端强大。
- 页面稳定，逻辑在后端演进。
- 小智负责速度，自建 AI 负责家庭智能。
- 手机负责移动性和复杂管理。
- 每个能力先定义安全工具，再让 AI/页面/手机调用。
- Page Agent First：一级页面内交互优先进入对应 Page Agent。
- Router Fallback：跨域或不明确意图再交给 Router。

## 非目标

- 不把圆屏做成手机、平板或完整 App 平台。
- 不把复杂业务塞进 ESP32。
- 不让圆屏承担后台管理。
- 不让 AI 裸调危险 API。
- 不把 Family Backend 绑定死到某一个语音服务实现。
- 不在当前阶段替换小智官方低延迟语音链路。
